Ejercicio entrega 29 de Agosto
Sistema de Gestión de una Biblioteca
Enunciado del Ejercicio
Desarrolla un sistema de gestión para una biblioteca que permita administrar diferentes tipos de materiales bibliográficos (libros, revistas, material audiovisual), usuarios, y préstamos. El sistema debe aplicar los cuatro pilares de la programación orientada a objetos:
Abstracción: Crear una clase abstracta MaterialBiblioteca que defina la estructura común para todos los materiales.
Encapsulamiento: Implementar atributos privados y métodos públicos para acceder y modificar estos atributos.
Herencia: Crear subclases específicas para diferentes tipos de materiales que hereden de la clase base.
Polimorfismo: Implementar métodos que se comporten de manera diferente según el tipo de material, como el cálculo de la fecha de 
devolución o la verificación de disponibilidad.
Explicación de los Cuatro Pilares en el Código
Abstracción:
La clase abstracta MaterialBiblioteca define la estructura común y comportamiento para todos los materiales.
Los métodos abstractos como calcular_fecha_devolucion(), obtener_tipo() y obtener_detalles() establecen una interfaz que todas las subclases deben implementar.
Encapsulamiento:
Todos los atributos están definidos como privados.
Se proporcionan métodos públicos (getters y setters) para acceder y modificar estos atributos de manera controlada.
La implementación interna está oculta y solo se expone la interfaz necesaria:
Herencia:
Las clases Libro, Revista y MaterialAudiovisual heredan de la clase base MaterialBiblioteca.
Cada subclase extiende la funcionalidad de la clase base añadiendo atributos y comportamientos específicos.
Polimorfismo:
El método calcular_fecha_devolucion() se implementa de manera diferente en cada subclase:
Libros: 15 días
Revistas: 7 días
Material audiovisual: 3 días
Los métodos obtener_tipo() y obtener_detalles() también tienen implementaciones específicas para cada tipo de material.
La biblioteca puede tratar a todos los materiales de manera uniforme a través de la interfaz común, sin importar su tipo específico.
Este sistema de gestión de biblioteca demuestra cómo los cuatro pilares de la programación orientada a objetos trabajan juntos para crear un diseño modular, extensible y mantenible.
